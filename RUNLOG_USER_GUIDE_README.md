# RunLog 软件操作说明书

本文档用于整理 RunLog App 的日常使用流程。发布新版本时，可以在此基础上补充截图、下载地址和版本说明。

## 1. 软件说明

RunLog 是 Android 原生跑步数据管理与运行测试工具。

本软件完全免费，如果你付费了说明你被骗了。

开源地址：

```text
https://github.com/hzjh22/yunrun-apk
```

## 2. 首次安装

1. 安装 APK。
2. 打开 RunLog。
3. 进入“我的”页面，先完成配置获取。
4. 进入“数据”页面，导入或提取历史跑步数据。
5. 回到“首页”，点击“检测配置”确认配置完整。
6. 点击“开始跑步”执行运行流程。

## 3. 页面说明

### 3.1 首页

首页用于开始和停止运行流程。

显示内容：

- 当前选中的跑步数据。
- 长度。
- 时间。
- 配速。
- 当前运行进度。
- 开始跑步按钮。
- 停止按钮。
- 检测配置按钮。

点击“开始跑步”后，软件会先检测配置和数据是否可用。配置不完整或没有历史跑步数据时，不会继续提交。

### 3.2 数据页

数据页用于管理历史跑步数据。

主要功能：

- 导入历史 JSON/TXT。
- 在线提取历史跑步数据。
- 查看已提取数据。
- 随机展示一组数据。
- 指定使用某条数据。
- 查看出现次数和使用次数。

数据卡片字段：

- 长度：跑步里程。
- 时间：跑步耗时。
- 使用：该条数据被指定或运行使用的次数。
- 出现：重复导入或提取时识别到的次数。
- 轨迹点：跑步轨迹点数量。
- 打卡点：任务打卡点数量。
- 配速：该数据的配速。

重复数据会自动合并，不会重复保存成多条。

### 3.3 我的中心

我的中心用于配置、日志和基础操作。

主要功能：

- 检测配置。
- 刷新 Token。
- 导入抓包配置。
- 查看公告。
- 切换随机/指定模式。
- 开关漂移。
- 清除历史数据。
- 查看日志文件路径和最近日志。

我的中心不会显示 `schoolHost`。

## 4. 配置获取方式

### 4.1 账号密码登录刷新 Token

进入“我的”页面，点击“刷新 TOKEN”。

需要填写：

- 学校名称。
- 账号。
- 密码。

登录成功后，软件会保存：

- Token。
- deviceId。
- deviceName。
- uuid。
- schoolId。
- schoolName。
- 必要的运行配置。

登录失败时，查看页面提示和最近日志。

### 4.2 导入抓包配置

进入“我的”页面，点击“导入配置”。

选择 PCAPdroid 导出的文本文件。软件会尝试从文本中提取：

- token。
- deviceId。
- deviceName。
- uuid。
- version。
- platform。
- 其他必要请求头。

导入后会自动检测配置完整性。配置不完整时会保存为草稿，不会直接用于开始跑步。

## 5. 历史数据获取方式

### 5.1 手动导入历史文件

进入“数据”页面，点击“导入历史”。

支持包含跑步详情的 JSON/TXT 文件。软件会扫描其中的跑步详情对象并保存到本地 SQLite。

### 5.2 在线提取历史数据

进入“数据”页面，点击“在线提取”。

该功能依赖当前配置和 Token。提取过程中会显示进度，失败时查看日志。

## 6. 跑步数据选择模式

### 6.1 随机模式

随机模式下，开始跑步时会从本地可用数据中随机选取一条。

数据页的“换一组”会随机展示 5 条数据，方便快速挑选。

### 6.2 指定模式

在数据页点击某条数据的“指定使用”后，软件会切换为指定模式。

指定模式下，首页会优先使用已指定的数据。

## 7. 开始跑步

开始前建议确认：

- 配置检测通过。
- Token 未失效。
- 已导入或提取历史跑步数据。
- 当前数据模式符合预期。

点击“开始跑步”后：

```text
检测配置 -> 选择数据 -> 读取任务 -> 执行运行逻辑 -> 显示进度 -> 写入日志
```

如果需要中断，点击“停止”。

## 8. 清除历史数据

进入“我的”页面，点击“清除历史数据”。

会清除：

- 本地配置。
- 登录参数。
- Token。
- 配置备份。
- 本地日志。
- 已提取或导入的历史跑步数据。

清除后需要重新导入配置和历史数据。

## 9. 日志查看

“我的”页面会显示：

- 日志文件路径。
- 最近日志内容。

遇到失败时，优先查看最近日志，确认是配置缺失、Token 失效、网络失败还是数据格式问题。

## 10. 常见问题

### 10.1 点击开始跑步后提示配置不完整

先进入“我的”页面点击“检测配置”。如果缺少 Token、deviceId、deviceName 或 uuid，重新登录或导入抓包配置。

### 10.2 在线提取失败

可能原因：

- Token 已失效。
- 当前配置不完整。
- 网络无法访问目标接口。
- 返回数据格式变化。

建议先刷新 Token，再重试在线提取。

### 10.3 数据页没有数据

先导入历史 JSON/TXT，或使用在线提取历史数据。

### 10.4 随机模式没有选中数据

确认数据页已经有可用数据。没有数据时随机模式不会生成任务。
